# 새 챕터 생성

새로운 챕터를 생성하고 프로젝트 구조에 통합합니다.

## 사용법
```
/new-chapter {챕터번호} {챕터제목} {설명}
```

## 실행 절차

### 1단계: 기존 구조 확인
- SUMMARY.md 읽기
- chapters/ 디렉토리의 기존 챕터 확인
- 챕터 번호 충돌 여부 검증

### 2단계: 디렉토리 및 파일 생성
- `chapters/{번호}-{슬러그}/README.md` 생성
- `chapters/{번호}-{슬러그}/assets/` 디렉토리 생성
- README.md에 챕터 제목, 개요, 학습 목표 템플릿 작성

### 3단계: SUMMARY.md 업데이트
- 적절한 위치에 새 챕터 링크 추가
- 기존 목차 순서 유지

### 4단계: 검증
- 생성된 파일 목록 출력
- SUMMARY.md 링크 유효성 확인

## 인자
- `$ARGUMENTS`: 챕터번호, 챕터제목, 설명을 공백으로 구분하여 전달
