# AI 강의 자료 프로젝트

## 프로젝트 개요
- **목적**: AI 관련 강의 자료를 책(GitBook) 형태로 작성·배포
- **대상**: AI/ML을 학습하려는 개발자 및 실무자
- **형식**: GitBook 기반 마크다운 콘텐츠

## 디렉토리 구조
```
lecture/
├── book.json            # GitBook 설정
├── SUMMARY.md           # 목차 (모든 챕터/섹션 링크)
├── README.md            # 책 소개 페이지
├── chapters/            # 챕터별 콘텐츠
│   └── {번호}-{슬러그}/
│       ├── README.md    # 챕터 메인
│       ├── {섹션}.md    # 하위 섹션
│       └── assets/      # 이미지, 다이어그램
├── assets/              # 공통 이미지/리소스
└── .gitignore
```

## 콘텐츠 작성 규칙

### 챕터 구조
- 챕터 디렉토리: `chapters/{2자리번호}-{영문슬러그}/` (예: `01-intro`, `02-prompt-engineering`)
- 챕터 메인: `README.md` — 챕터 개요, 학습 목표
- 섹션 파일: `{번호}-{슬러그}.md` (예: `01-what-is-ai.md`)
- 이미지: 해당 챕터의 `assets/` 디렉토리에 저장

### 마크다운 컨벤션
- 제목: `#` 하나만 사용 (페이지당 H1 하나)
- 코드 블록: 언어 태그 필수 (```python, ```bash 등)
- 팁/경고: GitBook hint 문법 사용 (`{% hint style="info" %}`)
- 이미지 경로: 상대 경로 사용 (`./assets/image.png`)

### 글쓰기 톤
- 존댓말 사용 (합쇼체가 아닌 해요체)
- 전문 용어는 첫 등장 시 설명
- 실습 예제 포함 권장
- 각 섹션 끝에 핵심 요약 포함

### SUMMARY.md 관리
- 새 챕터/섹션 추가 시 반드시 SUMMARY.md 업데이트
- 들여쓰기로 계층 구조 표현
- 링크가 실제 파일과 일치하는지 검증

## 품질 기준
- 오탈자 없음
- 코드 예제는 실행 가능해야 함
- 이미지에 alt 텍스트 포함
- 각 챕터는 독립적으로 읽을 수 있어야 함
- SUMMARY.md와 실제 파일 구조 일치
